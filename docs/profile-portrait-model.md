# 家庭画像与居民画像结论规范

本文定义家庭画像和居民画像的最终结论应覆盖哪些内容，以及哪些结论需要谨慎处理。它不限定算法实现方式，也不要求算法必须输出证据链、置信度或中间推理过程。

本规范关注的是最终自然语言报告或结构化结果中的“结论质量”：报告说了什么、覆盖了哪些角度、有没有明显越界。

## 1. 基本原则

1. 画像结论应描述家庭和居民的长期生活模式，而不是复述单条事件。
2. 家庭画像描述“这个家如何运行”，居民画像描述“匿名居民或成员槽位如何生活”。
3. 居民画像默认不绑定真实身份，除非输入或业务场景明确提供了身份绑定。
4. 对身份、角色、健康、关系、精确轨迹等敏感结论应保持谨慎。
5. 不能判断的内容可以不输出，不应为了完整性编造结论。

## 2. 家庭画像应包含的结论

家庭画像回答的是：这个家整体如何运转。

| 类别 | 应关注的结论 |
| --- | --- |
| 家庭结构 | 家庭规模倾向、是否可能存在多个成员槽位、儿童/老人/宠物等候选角色 |
| 房屋与空间 | 房间构成、公共区/私密区、入口、主要活动区域 |
| 空间使用 | 各房间主要用途、活跃时间段、多用途空间 |
| 生活节律 | 起床、离家、返家、晚餐、洗澡、睡眠等 routine，工作日/周末差异 |
| 活动模式 | 做饭、用餐、睡眠、工作学习、娱乐、清洁、洗衣、出入户 |
| 设备习惯 | 常用设备、设备组合、典型使用顺序、手动控制倾向 |
| 舒适偏好 | 温度、湿度、灯光、窗帘、空气质量等家庭级偏好 |
| 能耗与资源 | 用电、用水、空调、热水器、厨房设备等使用习惯 |
| 家务与维护 | 清洁、洗衣、垃圾处理、耗材更换、设备维护 |
| 异常与变化 | 与日常模式不同的变化、季节性差异、短期状态变化 |
| 自动化机会 | 适合提醒、推荐或自动化的低风险场景 |

家庭画像不要求每份报告覆盖所有类别。数据不足或结论价值不高时，可以省略。

## 3. 居民画像应包含的结论

居民画像回答的是：某个匿名居民或成员槽位通常如何生活、偏好什么、可能需要什么。

| 类别 | 应关注的结论 |
| --- | --- |
| 匿名槽位 | 可能存在的居民槽位、槽位数量倾向、槽位稳定性 |
| 作息模式 | 起床、睡觉、离家、回家、午休、周末节奏 |
| 空间偏好 | 常待房间、强相关房间、公共区/私密区使用倾向 |
| 设备偏好 | 常用设备、手动控制习惯、设备组合 |
| 活动偏好 | 做饭、运动、阅读、看电视、学习、远程办公、家务等活动倾向 |
| 家庭 routine 关系 | 该槽位可能参与哪些家庭 routine，哪些模式无法归属个人 |
| 舒适偏好 | 温度、灯光、窗帘、空气质量、湿度等个人化倾向 |
| 出行模式 | 离家和回家时间窗口、周末外出倾向 |
| 自动化接受度 | 是否适合提醒、推荐或自动执行，是否可能经常手动覆盖 |
| 家务分工倾向 | 厨房、洗衣、清洁、垃圾处理等活动归属倾向 |
| 提醒与服务需求 | 免打扰窗口、提醒方式、低风险照护或安全提醒 |
| 变化趋势 | 作息、空间使用、设备使用、外出频率等变化 |

居民画像应优先使用匿名槽位，例如“偏好书房的槽位”“晚间厨房活动槽位”“儿童活动候选槽位”。只有在输入中已经明确绑定身份时，才适合输出具体成员画像。

## 4. 自然语言报告建议结构

一份自然语言画像报告可以按以下结构组织：

1. 家庭整体摘要。
2. 空间与房间使用。
3. 生活节律与主要 routine。
4. 匿名居民槽位和可能个人 routine。
5. 设备习惯与自动化机会。
6. 不确定或需要确认的结论。

报告应尽量输出有信息量的结论，避免两种情况：

- 只复述事件流水，没有形成画像。
- 把无法判断的内容写成确定事实。

## 5. 谨慎结论

以下结论可以作为候选方向，但不应轻易写成确定事实：

| 类型 | 示例 |
| --- | --- |
| 家庭人数 | “三口之家”“独居”“多人同住” |
| 年龄角色 | “儿童”“老人”“学生”“退休人员” |
| 职业身份 | “远程办公人员”“家庭主妇”“保姆” |
| 宠物 | “家里养宠物” |
| 照护关系 | “有人被照护”“有人承担照护责任” |
| 健康状态 | 睡眠问题、行动不便、疾病、异常健康风险 |
| 亲密关系 | 伴侣关系、家庭关系变化 |
| 精确轨迹 | 具体个人的精确出行路线或长期位置追踪 |

这类结论更适合表达为“可能存在”“候选”“倾向于”，或放入“需要确认”的部分。

## 6. 不应输出的结论

画像报告不应输出：

- 未授权的真实姓名、身份、职业、年龄、性别、家庭关系绑定。
- 健康、宗教、财务、亲密关系等敏感判断。
- 从一次或少量事件推出长期习惯。
- 从房间名直接推出居民身份，例如“儿童房有活动，所以家里有儿童”。
- 从设备数量、房间数量直接推出家庭人数。
- 把家庭整体 routine 直接写成某个具体人的 routine。
- 无法解释适用场景的高影响自动化建议。

## 7. 判断报告好坏的重点

比较不同算法的最终报告时，重点看：

1. 是否覆盖了家庭画像和居民画像的核心角度。
2. 是否区分家庭层结论和居民槽位结论。
3. 是否能总结长期模式，而不是复述事件。
4. 是否能表达不确定性，而不是强行下结论。
5. 是否避免身份、角色和敏感信息的过度推断。
6. 是否提出有实际价值的提醒、推荐或自动化机会。

一个好的画像报告不一定有统一格式，也不一定输出证据链或置信度；但它应该覆盖关键生活维度，结论克制，边界清楚，并且对用户或业务有用。
