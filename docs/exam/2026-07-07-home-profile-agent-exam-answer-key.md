# 家庭画像 Agent 作答试卷参考答案

本文是 `2026-07-07-home-profile-agent-exam.md` 的参考答案和评分要点。评分时不要求逐字匹配，只看是否命中关键方向、是否避免明显越界。

每题 3 分，总分 60 分。

## A. 家庭结构与成员槽位

### Q1. 家庭组成

满分要点：

- 家庭整体可判断为三口之家加一只宠物。
- 包含两位成年人、一名学生儿童、一只宠物。
- 可以表达为“家庭规模倾向”或“成员槽位”，不要求真实身份。

扣分点：

- 把老人加入当前家庭。
- 完全忽略宠物。
- 只说“多人家庭”，没有成员结构。

### Q2. 是否存在老人

满分要点：

- 当前不应判断家中存在老人。
- 标准画像中没有启用老人；即使系统模板可能存在 senior persona，也不应计入当前家庭画像。
- 如果只能基于事件，也应回答“没有足够信号支持老人结论”。

扣分点：

- 明确断言家中有老人。
- 因为白天有人在家就推断老人。

### Q3. 主要居民槽位

满分要点：

- 通勤成年人：偏早起，参与早餐、通勤离家、傍晚回家、安全门锁相关行为。
- 白天在家办公成年人：工作日白天主要在书房，远程/混合办公，中午短暂到厨房，参与晚餐准备。
- 学生儿童：工作日早晨上学，傍晚回家后作业或客厅活动，晚上进入儿童房睡觉。
- 宠物槽位：主要活动在客厅、厨房、花园，夜间较安静。

扣分点：

- 把所有活动归到一个成年人。
- 给出真实姓名或未授权身份。
- 漏掉儿童或远程办公槽位。

### Q4. 是否能归为同一人

满分要点：

- 不应把所有家庭活动归属于同一个人。
- 至少应拆分通勤成年人、白天在家办公成年人、儿童活动槽位、宠物活动槽位。
- 理由是时间、空间和活动模式明显不同：早晚通勤、白天书房、儿童房睡前、花园/宠物活动等。

扣分点：

- 只输出一个家庭成员或一个主用户。
- 把家庭 routine 直接写成某个具体人的 routine。

## B. 工作日与周末生活节律

### Q5. 工作日早晨流程

满分要点：

- 大约 6 点多起床。
- 起床后有洗漱、早餐、出门或上学流程。
- 早餐通常在 6:30-7:30。
- 儿童上学离家，通勤成年人离家。

扣分点：

- 说工作日早晨无明显 routine。
- 把早餐说成稳定热餐或周末 brunch。

### Q6. 工作日白天活动

满分要点：

- 工作日白天家中有稳定活动。
- 主要在书房，指向远程/混合办公成年人。
- 中午会离开书房到厨房短暂午餐，然后返回书房。

扣分点：

- 判断白天全员离家。
- 把白天在家直接推成老人。

### Q7. 工作日晚间 routine

满分要点：

- 傍晚有儿童回家和作业/客厅活动。
- 18 点后有稳定晚餐准备。
- 晚餐准备约 45 分钟后全家用餐。
- 晚饭后转到客厅看电视或共处。
- 儿童约 21 点进入睡前流程，成年人约 22 点后入睡。

扣分点：

- 漏掉晚餐烹饪或家庭共处。
- 把儿童睡前和成年人睡眠时间混淆。

### Q8. 周末节奏

满分要点：

- 周末起床比工作日晚。
- 周末早餐更像 cooked brunch，不是 quick cold breakfast。
- 非雨天周末下午可能全家外出，傍晚返回。
- 雨天周末更可能客厅家庭娱乐。
- 周末睡觉时间相对更晚。

扣分点：

- 把周末和工作日早餐混为一谈。
- 完全没有提到天气对周末活动的影响。

## C. 空间使用画像

### Q9. 书房功能

满分要点：

- 书房是稳定功能空间，不是偶发活动空间。
- 工作日白天主要由白天在家办公成年人使用。
- 可表述为远程办公或工作/学习强信号空间。

扣分点：

- 只说书房偶尔有人活动。
- 直接把书房活动断言为学生行为。

### Q10. 厨房功能

满分要点：

- 工作日早餐是 quick cold 风格，主要涉及冰箱和厨房灯，不一定使用灶台。
- 工作日晚餐是稳定烹饪场景。
- 晚餐通常由白天在家办公成年人准备。
- 周末早午餐使用灶台，但强度和时长低于晚餐。

扣分点：

- 把所有早餐都说成热餐。
- 漏掉工作日晚餐稳定烹饪。

### Q11. 客厅、儿童房、主卧

满分要点：

- 客厅：家庭共处、娱乐、儿童作业，傍晚和周末活跃。
- 儿童房：儿童睡前和睡眠空间，晚间信号明显。
- 主卧：成年人睡眠空间。
- 应避免仅因房间名就强推身份，但在本数据中与行为模式共同支持这些空间功能。

扣分点：

- 混淆儿童房和主卧。
- 完全忽略客厅家庭互动。

### Q12. 花园解释

满分要点：

- 花园与季节性喷淋、短时查看、宠物活动相关。
- 夏季清晨有喷淋特征。
- 花园或摄像头活动不应直接解释为外部入侵。

扣分点：

- 把宠物活动高置信解释为入侵。
- 完全忽略花园的季节性维护。

## D. 设备、家务与使用习惯

### Q13. 厨房设备联动

满分要点：

- 晚餐准备时冰箱会短暂打开后关闭。
- 灶台开启时油烟机同步开启。
- 灶台关闭后油烟机关闭。
- 这是稳定做饭链路，尤其适合晚餐场景。

扣分点：

- 没有识别灶台和油烟机联动。
- 把冰箱打开直接解释为完整做饭。

### Q14. 扫地机器人和洗衣

满分要点：

- 扫地机器人通常在工作日通勤离家后约 10 分钟启动。
- 洗衣不是每天发生，约每两天一次。
- 洗衣通常由白天在家的成年人触发。

扣分点：

- 说扫地机器人每天任意时间运行。
- 说洗衣每天发生。

### Q15. 门锁事件

满分要点：

- 出门、回家、周末外出通常有 unlock -> lock 行为。
- 门锁可支持离家/回家和安全习惯判断。
- 单次门锁事件不能单独断言全家离家或全家到家。
- 只有解锁没有上锁应作为异常或低置信缺口。

扣分点：

- 单个解锁事件就判断全员到家。
- 忽略解锁后上锁的安全习惯。

### Q16. 环境传感器和空调信号

满分要点：

- 环境传感器持续运行，应作为环境上下文或周期上报，不等同人为操作。
- 空调目标温度围绕舒适温度，标准为约 25 摄氏度。
- 空调 target 或 mode 变化不一定是用户手动控制。
- 可以用于家庭舒适偏好，但不应过度解释为具体个人行为。

扣分点：

- 把所有环境传感器事件都当成人类活动。
- 把空调变化高置信归因给某个具体居民。

## E. 安全、自动化与服务机会

### Q17. 高价值自动化推荐

满分要点：

- 推荐晚餐厨房安全联动。
- 触发条件：灶台功率升高或烹饪开始。
- 动作：开启油烟机；监控烟雾、PM2.5、灶台功率和厨房存在信号。
- 灶台关闭后延迟关闭油烟机。
- 风险提醒：灶台长时间开启但厨房无人，或烟雾/空气质量异常时提醒。

扣分点：

- 推荐高影响但无条件约束的自动执行。
- 推荐与该家庭模式无关的自动化。

### Q18. 安全风险或异常提醒

满分要点：

- 厨房用火风险，尤其灶台长时间开启或厨房无人。
- 水流长时间不关闭。
- 门锁解锁后未上锁。
- 夜间异常活动。
- 烟雾或空气质量异常。

扣分点：

- 完全没有厨房安全。
- 把所有花园摄像头活动都当成入侵报警。

### Q19. 宠物或花园摄像头

满分要点：

- 不应直接判断为安全入侵。
- 宠物在客厅、厨房和花园活动是合理解释。
- 花园还有季节性喷淋和短时查看等正常活动。
- 可作为低置信安全候选，但不应高置信报警。

扣分点：

- 高置信输出外部入侵。
- 完全忽略宠物解释。

## F. 边界与不可判断题

### Q20. 不应高确定性输出的结论

满分可列出以下任意 5 项以上，并说明不应高确定性的原因：

- 家中有老人。
- 宠物移动等同于安全入侵。
- 环境传感器上报等同于人为操作。
- 空调 target 或 mode 变化一定是用户手动控制。
- 单次门锁事件足以断言全家离家或全家到家。
- 单日或两日数据足以确认长期趋势。
- 从房间名直接确认儿童身份。
- 从设备数量或房间数量直接确认家庭人数。
- 把所有家庭活动归到同一个人。
- 健康、宗教、财务、亲密关系等敏感推断。

扣分点：

- 少于 5 项。
- 列出的内容大多与本家庭画像无关。
- 把标准答案中的强结论也全部说成不可判断。

## 总分解释

- 50-60 分：能较完整支持家庭画像和居民画像问答，边界意识较好。
- 40-49 分：主要家庭模式可用，但居民槽位或边界题仍有缺口。
- 30-39 分：能回答部分 routine，但画像结构和应用价值不足。
- 20-29 分：多为零散事实，难以支持稳定画像。
- 0-19 分：核心家庭结构和生活模式明显错误，或存在严重越界。
